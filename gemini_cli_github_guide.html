Gemini CLI를 이용한 GitHub 입문 가이드

이 가이드는 프로그래밍 초보자가 Git과 GitHub의 기본 개념을 이해하고, Gemini CLI를 사용하여 코드를 관리하는 방법을 배우는 것을 목표로 합니다.

Part 1: GitHub 계정 만들기

코드를 세상과 공유하고 다른 개발자와 협업하기 위한 첫 단계입니다.

  1. GitHub 가입 페이지로 이동합니다.
  2. 원하는 사용자 이름, 이메일 주소, 비밀번호를 입력하고 화면의 안내에 따라 계정 생성을 완료합니다.
  3. 가입 시 입력한 이메일 주소로 인증 메일이 발송됩니다. 메일을 확인하여 인증을 완료해야 합니다.

Part 2: 로컬 개발 환경 설정하기

이제 내 컴퓨터에 프로젝트를 저장할 공간을 만들고 Git으로 관리를 시작하겠습니다. 모든 명령어는 Gemini CLI를 통해 실행합니다.

1. 프로젝트 폴더 생성

my-first-repo라는 이름의 폴더를 만듭니다.

mkdir my-first-repo

2. Git 저장소 초기화

방금 만든 폴더로 이동하여 Git이 이 폴더를 관리하도록 설정합니다. 이 과정을 '저장소(Repository) 초기화'라고 합니다.

cd my-first-repo
git init

무슨 일이 일어났나요? git init 명령어는 폴더 내에 .git이라는 숨김 폴더를 만듭니다. 이 폴더에는 변경 이력, 설정 등 Git이 프로젝트를 관리하는데 필요한 모든 정보가 저장됩니다.

Part 3: GitHub에 원격 저장소 만들기

GitHub에 저장소를 만들면 "…or push an existing repository from the command line" 섹션에 몇 가지 명령어가 표시됩니다. 이 명령어를 사용하여 로컬과 원격을 연결합니다.

  1. GitHub에 로그인합니다.
  2. 오른쪽 상단의 '+' 아이콘을 클릭하고 'New repository'를 선택합니다.
  3. Repository name에 로컬 폴더와 같은 이름(my-first-repo)을 입력합니다.
  4. 다른 설정(Description, Public/Private)은 자유롭게 선택하되, "Initialize this repository with a README"는 절대 체크하지 마세요. 비어있는 저장소를 만들어야 로컬 저장소와 쉽게 연결할 수 있습니다.
  5. 'Create repository' 버튼을 클릭합니다.

Part 4: 로컬 저장소와 원격 저장소 연결하기

GitHub에 저장소를 만들면 "…or push an existing repository from the command line" 섹션에 몇 가지 명령어가 표시됩니다. 이 명령어를 사용하여 로컬과 원격을 연결합니다.

1. 원격 저장소 주소 추가

내 로컬 저장소에 'origin'이라는 이름으로 GitHub 저장소의 주소를 알려줍니다.

# 아래 URL은 본인의 GitHub 저장소 URL로 변경해야 합니다.
git remote add origin https://github.com/YOUR_USERNAME/my-first-repo.git

'origin'이란? 'origin'은 원격 저장소 주소를 가리키는 별명입니다. 다른 이름으로 변경할 수 있지만, 전 세계 개발자들이 관습적으로 'origin'을 사용합니다.

2. 기본 브랜치 이름 변경

요즘은 기본 브랜치 이름으로 'main'을 사용하는 것이 표준입니다.

git branch -M main

3. 연결 확인

아래 명령어로 원격 저장소가 잘 연결되었는지 확인합니다.

git remote -v

fetch와 push 주소가 모두 표시되면 성공입니다.

Part 5: Push - 내 코드를 GitHub에 올리기

이제 로컬에서 파일을 만들고, 변경 이력을 기록(Commit)한 뒤, GitHub에 올려보겠습니다(Push).

1. 파일 생성

Gemini CLI의 write_file 기능을 사용하여 프로젝트에 대한 간단한 소개 파일 README.md를 만듭니다.

write_file "README.md" "# My First Repository\n\nThis is my first project managed with Git and Gemini CLI."

2. 변경사항 스테이징 (Staging)

Git에게 어떤 파일을 변경 이력에 포함시킬지 알려주는 단계입니다. 장바구니에 물건을 담는 것과 비슷합니다.

git add README.md

3. 변경사항 커밋 (Commit)

스테이징된 파일들을 하나의 의미있는 변경 단위(스냅샷)로 묶고, 어떤 변경사항인지 메시지를 남깁니다.

git commit -m "feat: Create initial README file"

커밋 메시지: -m 뒤에 붙는 메시지는 이 변경사항에 대한 요약입니다. 다른 사람(그리고 미래의 나)이 이해하기 쉽도록 명확하게 작성하는 습관이 중요합니다.

4. GitHub에 푸시 (Push)

로컬에 저장된 커밋들을 'origin'이라는 이름의 원격 저장소(GitHub)의 'main' 브랜치로 밀어 올립니다.

git push -u origin main

이제 GitHub 저장소 페이지를 새로고침 해보세요. 방금 만든 README.md 파일이 보일 겁니다!

Part 6: Pull - GitHub의 변경사항을 내 컴퓨터로 가져오기

다른 사람이 코드를 변경했거나, 내가 다른 컴퓨터에서 작업한 내용을 현재 컴퓨터로 가져올 때 사용합니다.

1. GitHub에서 파일 직접 수정하기

GitHub 저장소 페이지에서 README.md 파일을 클릭하고, 연필 모양(Edit) 아이콘을 눌러 내용을 수정한 뒤 'Commit changes' 버튼을 눌러 저장합니다.

2. 원격 변경사항 가져오기 (Pull)

이제 원격 저장소(GitHub)는 변경되었지만 내 로컬 컴퓨터는 아직 옛날 버전입니다. pull 명령어로 최신 변경사항을 가져옵니다.

git pull origin main

3. 변경사항 확인

Gemini CLI의 read_file 기능으로 README.md 파일의 내용이 GitHub에서 수정한 대로 바뀌었는지 확인합니다.

read_file "README.md"

축하합니다! 이제 여러분은 Gemini CLI를 사용하여 GitHub의 가장 핵심적인 기능인 push와 pull을 모두 할 수 있게 되었습니다.

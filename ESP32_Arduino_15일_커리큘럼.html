UTTEC Lab AI 보드를 위한 ESP32 아두이노 15일 완성 커리큘럼

UTTEC Lab AI 교육용 보드의 모든 기능을 아두이노(Arduino) 환경에서 마스터할 수 있도록 설계된 15일 과정입니다. C언어나 임베디드 시스템에 대한 사전 지식이 없는 초보자를 대상으로 합니다.

📝 교육 과정 개요

주차 일차 주요 학습 내용 목표
1주차 1-5일 기초 다지기: 아두이노 IDE 설정, 기본 문법, 디지털 입출력 개발 환경을 구축하고 LED, 버튼 등 간단한 부품을 제어합니다.
2주차 6-10일 핵심 부품 활용: 버저(소리), I2C 통신, OLED 디스플레이, 온습도 센서 보드의 핵심 센서와 디스플레이를 다루는 방법을 배웁니다.
3주차 11-15일 고급 기능 및 프로젝트: 기압 센서, 와이파이(WiFi) 통신, 종합 프로젝트 인터넷에 연결하고, 지금까지 배운 모든 기능을 활용한 나만의 IoT 장치를 만듭니다.

📅 일자별 상세 커리큘럼

1주차: 아두이노와 친해지기

1일차: 첫 만남, 개발 환경 설정

  • 주제: 아두이노 IDE 설치 및 ESP32 보드 설정
  • 목표: 개발 환경을 구축하고, 보드가 컴퓨터와 정상적으로 통신하는지 확인합니다.
  • 핵심 개념: IDE, 보드 매니저, 라이브러리, 시리얼 포트, 컴파일, 업로드
  • 실습:
    1. 아두이노 IDE 설치
    2. ESP32 보드 개발 환경 추가
    3. 예제 > 01.Basics > Blink 코드를 업로드하여 내장 LED 깜박이기

2일차: 디지털 출력 - LED 제어

  • 주제: GPIO 디지털 출력으로 3색 LED 켜고 끄기
  • 목표: 코드에서 보드의 빨강, 노랑, 파랑 LED를 원하는 대로 제어할 수 있습니다.
  • 핵심 개념: pinMode(), digitalWrite(), delay(), HIGH, LOW, GPIO 핀 번호
  • 실습:
    1. 3개 LED를 1초마다 순서대로 켜는 코드 작성
    2. 신호등(빨강-노랑-파랑) 패턴 만들기

3일차: 디지털 입력 - 버튼 상태 읽기

  • 주제: GPIO 디지털 입력으로 버튼(Switch) 누름 감지하기
  • 목표: 버튼을 눌렀을 때 특정 동작을 수행하는 코드를 작성할 수 있습니다.
  • 핵심 개념: digitalRead(), if/else 조건문, 풀업/풀다운 저항
  • 실습:
    1. 버튼을 누르고 있는 동안 LED 켜기
    2. 버튼을 한 번 누르면 LED가 켜지고, 다시 누르면 꺼지는 토글(Toggle) 기능 구현

4일차: 시리얼 통신 - 컴퓨터와 대화하기

  • 주제: 시리얼 모니터를 이용한 디버깅 및 데이터 확인
  • 목표: 보드의 상태나 변수 값을 컴퓨터 화면에 출력하여 코드의 동작을 쉽게 확인할 수 있습니다.
  • 핵심 개념: Serial.begin(), Serial.print(), Serial.println()
  • 실습:
    1. 버튼이 눌렸을 때 "Button Pressed!" 메시지 출력하기
    2. LED가 켜지거나 꺼질 때 현재 상태를 시리얼 모니터에 표시하기

5일차: 1주차 리뷰 - 미니 프로젝트

  • 주제: 버튼과 LED를 이용한 간단한 상호작용 프로그램 만들기
  • 목표: 디지털 입출력과 시리얼 통신을 종합적으로 활용합니다.
  • 실습: 버튼을 누를 때마다 빨강 -> 노랑 -> 파랑 -> 꺼짐 순으로 LED 상태가 변경되고, 현재 켜진 LED 색상을 시리얼 모니터에 출력하는 프로그램을 만듭니다.

2주차: 센서와 디스플레이 활용

6일차: 아날로그 출력(PWM) - 소리 만들기

  • 주제: 버저(Buzzer)를 이용하여 소리내기
  • 목표: tone() 함수로 다양한 높낮이의 소리를 만들어 멜로디를 연주할 수 있습니다.
  • 핵심 개념: PWM, 주파수(Hz), tone(), noTone()
  • 실습:
    1. '도-레-미' 음계 연주하기
    2. 버튼을 누르면 '삐' 소리가 나게 하기
    3. 간단한 동요('반짝반짝 작은 별') 멜로디 연주하기

7일차: I2C 통신 - 장치 간의 대화법

  • 주제: I2C 프로토콜의 이해와 I2C 주소 스캔
  • 목표: 보드에 연결된 여러 I2C 장치(OLED, 센서)를 식별하고 통신 준비를 합니다.
  • 핵심 개념: I2C, SDA/SCL, 주소(Address)
  • 실습: I2C 스캐너 코드를 실행하여 OLED(0x3C), AHT20(0x38), BMP280(0x76)의 주소를 시리얼 모니터에서 직접 확인합니다.

8일차: I2C 디스플레이 - OLED에 정보 표시

  • 주제: SSD1306 OLED 디스플레이 사용법
  • 목표: OLED에 원하는 텍스트, 숫자, 간단한 도형을 출력할 수 있습니다.
  • 필요 라이브러리: Adafruit_SSD1306, Adafruit_GFX
  • 실습:
    1. "Hello, UTTEC!" 메시지 출력하기
    2. 버튼을 누른 횟수를 OLED 화면에 표시하기

9일차: I2C 센서(1) - 온습도 측정

  • 주제: AHT20 센서로 현재 온도와 습도 읽기
  • 목표: 주변 환경의 온도와 습도 데이터를 측정하여 시리얼 모니터에 출력합니다.
  • 필요 라이브러리: Adafruit_AHTX0
  • 실습:
    1. 1초마다 현재 온도(°C)와 상대 습도(%)를 시리얼 모니터에 출력하기

10일차: 2주차 리뷰 - 미니 프로젝트

  • 주제: 온습도계 만들기
  • 목표: 센서와 디스플레이를 결합하여 실용적인 장치를 만듭니다.
  • 실습: AHT20 센서에서 측정한 현재 온도와 습도 값을 OLED 디스플레이에 실시간으로 표시하는 디지털 온습도계를 만듭니다.

3주차: IoT 세상으로 나가기

11일차: I2C 센서(2) - 기압 측정

  • 주제: BMP280 센서로 현재 기압과 고도 읽기
  • 목표: 기압 데이터를 측정하고, 이를 이용해 현재 고도를 추정할 수 있습니다.
  • 필요 라이브러리: Adafruit_BMP280
  • 실습:
    1. 현재 기압(hPa)과 온도(°C)를 시리얼 모니터에 출력하기
    2. 기압을 이용해 계산된 추정 고도를 OLED 디스플레이에 표시하기

12일차: WiFi 연결 - 인터넷 세상으로

  • 주제: ESP32를 WiFi 공유기에 연결하기
  • 목표: 보드를 인터넷에 연결하고 할당된 IP 주소를 확인합니다.
  • 핵심 개념: WiFi SSID, 비밀번호, IP 주소
  • 필요 라이브러리: WiFi.h
  • 실습:
    1. 집이나 사무실의 WiFi에 보드를 연결하기
    2. 연결 성공 시 할당받은 IP 주소를 시리얼 모니터와 OLED에 출력하기

13일차: 웹 서버 - 나만의 웹페이지 만들기

  • 주제: ESP32로 간단한 웹 서버 만들기
  • 목표: 스마트폰이나 PC의 웹 브라우저로 ESP32에 접속하여 정보를 확인할 수 있습니다.
  • 핵심 개념: 웹 서버, 클라이언트, HTML
  • 실습:
    1. 웹 브라우저에 접속했을 때 "Welcome to UTTEC Lab AI" 메시지를 보여주는 웹 서버 만들기
    2. 웹 페이지에 현재 LED의 켜짐/꺼짐 상태를 표시하기

14일차: 최종 프로젝트 구상

  • 주제: "IoT 스마트 환경 모니터" 프로젝트 설계
  • 목표: 지금까지 배운 모든 기능을 종합하여 만들 최종 프로젝트의 기능과 동작 방식을 구체적으로 설계합니다.
  • 프로젝트 기능:
    • 온/습도, 기압 데이터를 주기적으로 측정
    • 측정된 데이터를 OLED 디스플레이에 표시
    • WiFi에 연결하여 웹 서버 기능 제공
    • 웹 브라우저 접속 시 현재 모든 센서 값을 보여줌
    • 특정 온도 이상일 경우 빨간 LED와 버저로 경고 알림

15일차: 최종 프로젝트 구현

  • 주제: "IoT 스마트 환경 모니터" 완성하기
  • 목표: 설계한 프로젝트를 코드로 완성하고 실제 동작을 테스트합니다.
  • 실습: 14일차에 설계한 모든 기능을 구현하고, 코드를 완성하여 나만의 IoT 환경 모니터를 만듭니다. 스마트폰으로 웹 서버에 접속하여 데이터가 정상적으로 보이는지 확인하고, 온도가 높을 때 경고가 잘 울리는지 테스트합니다.
